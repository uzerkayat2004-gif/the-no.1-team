# Pin and Send Back

## When It Happens
- Output doesn't match brainstorm decisions
- Code deviates from agreed design
- Research misses critical requirements
- Any task type: coding, design, research

## Process
1. Boss or team identifies the issue
2. Agent is **pinned** 📌 (status changes to "Pinned")
3. Specific feedback provided: what's wrong, what to fix
4. Agent returns updated output
5. Team reviews again

## Limits
- **Maximum 3 send-backs** per issue
- After 3 failed fixes → escalate to Boss
- Boss decides: reassign to different agent or resolve directly

## Status
- When pinned: agent status shows 📌 Pinned
- Agent must address ALL feedback points before resubmitting
- Must note what changed and why in their response
